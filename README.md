# paper2-isnv-transport

Filter transportability diagnostic for *Mycobacterium tuberculosis* intra-host
single nucleotide variant (iSNV) detection. Companion code to:

> Séraphin MN, et al. *Can a calibrated intra-host variant detection filter
> transfer across sequencing depth regimes? A transport diagnostic for*
> Mycobacterium tuberculosis *genomic epidemiology*.
> (in preparation, 2026).

This repository contains the Paper 2a pipeline. Paper 2b
(Edwards-2023 operationalization, HIV-iSNV worked example) lives in a
sibling branch/repo (see `README_split.md`).

## What this pipeline does

A replicate-anchored iSNV detection filter ladder (Looser / Primary / Tighter)
calibrated on Ghana same-visit replicate pairs (Séraphin et al., 2026, Paper 1)
is re-applied to a Florida tuberculosis surveillance archive sequenced at lower
mean depth. Two pre-specified diagnostics evaluate transport:

- **T1 (per-specimen prevalence band)**: Does Florida's Primary-tier detection
  rate fall within a pre-specified band around Ghana's per-patient anchor?
- **T2 (within-cluster Jaccard versus depth)**: Within Florida transmission
  clusters, does pair-level Jaccard rise with minimum pair depth, matching the
  Ghana within-host replicate-pair signal?

A post-hoc hurdle decomposition (any iSNV sharing + amount of iSNV sharing) localizes the
depth-detection coupling. eAppendix scripts produce supplementary tables and
figures (depth-floor derivation, leave-one-cluster-out sensitivity, both-empty
LPM-versus-logit scale check).

## Headline result

- T1: Florida Primary detection 26.61% (Wilson 95% CI 22.20-31.53%); pre-spec
  band [6.5%, 26.5%]; failed at the boundary by 0.1 pp.
- T2: Jaccard slope +0.0020 per +10x (HC1 cluster-robust 95% CI +0.0004 to
  +0.0035; p = 0.012); criterion met, same direction as Ghana anchor.
- Hurdle: depth-detection coupling concentrates at the extensive margin
  (P(Jaccard > 0 | union > 0)).

See `outputs/paper2a/tables/table2_transport_summary.csv` after running the
pipeline.

## Running the pipeline

Requires R 4.5.1 with packages: `sandwich`, `lmtest`, `MASS`, `quadprog`,
`ggplot2`, `dplyr`, `patchwork` (optional).

From the project root:

```r
source("R/run_paper2a.R")
```

or from the shell:

```sh
Rscript R/run_paper2a.R
```

The orchestrator runs:

1. Shared infrastructure (00 prep, 01 frames, 02 filter ladder).
2. Primary analyses (05 T1, 06 T2 Jaccard slope, 06b hurdle).
3. eAppendix analyses (06c LPM-vs-logit, 06d depth appropriateness,
   06e eTables S2-S3).
4. Manuscript deliverables (13a tables 1-2, 14a figures 1-2).

Outputs land under `outputs/paper2a/{tables,figures}/`. A timestamped log
lands in `outputs/orchestrator_logs/`.

Phase 1 and Phase 2 halt on error. Phase 3 and Phase 4 log-and-continue so a
single eAppendix-script failure does not block the main tables and figures.

## Reproducibility

- Random seed locked in `R/00_pipeline_config.R` (`SEED = 20260509`).
- Inherited Paper 1 thresholds asserted at load time in
  `validate_inherited_thresholds()`; pipeline halts on threshold drift.
- All cluster-robust standard errors computed with `sandwich::vcovCL` (HC1).
- Software versions are recorded in the orchestrator log via the banner.

## Data availability

Variant call sets and metadata necessary to reproduce the analyses are
deposited under NCBI BioProject PRJNA1466981. Derived data
files (`data_derived/`) are regenerated by the pipeline. The repository does
not redistribute raw sequencing reads.
